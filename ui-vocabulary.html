  • Hierarchy

    Primary Button

    The single most important action on a screen. Use once per view to anchor user intent and drive conversion. Filled, high contrast. Hover here reveals the gold slide-in animation.

  • Hierarchy

    Secondary Button

    Supports the primary action without competing with it. Used for alternate paths in dialogs and forms where two choices exist side by side.

  • Hierarchy

    Ghost / Text Button

    Lowest visual weight no background, no border. For tertiary actions and inline links, or in dense lists where full buttons would create visual clutter.

  • Semantics

    Destructive Button

    Signals irreversible or high-risk actions. The distinct color primes users to pause and read carefully. Always follow with a confirmation step for permanent operations.

  • State

    Disabled Button

    Indicates the action is currently unavailable. Reduces opacity rather than hiding preserving affordance and spatial layout. Always explain why the action is unavailable when possible.

  • Scale

    Small / Medium / Large

    Size communicates context and importance. Small (36px) for toolbars and dense UIs; medium (48px) for standard forms; large (56px) for hero CTAs and marketing pages where emphasis demands space.

  • Shape

    Pill Shape

    Fully rounded ends (border-radius: 9999px) create a softer, friendlier tone. Common in social actions, filter chips, and tags typically contrasting with a square-corner design system.

  • Pattern

    Icon Button

    Square button containing only an icon. Requires an aria-label no icon is self-explanatory to every user. Used in toolbars and compact UIs where text labels would crowd the layout.

  • Layout

    Full-Width / Block Button

    Stretches to fill its container (width: 100%). Essential for mobile checkout flows and single-column forms where generous touch targets and dominant CTAs are required.

Hover Animations

Slide-in — left

The gold fill translates in from the left on hover. Directional, cinematic. Feels like a reveal rather than a change.

Sweep — bottom up

Fill rises from the bottom edge. Creates a lifting motion that draws the eye upward, reinforcing forward action.

Wipe — right

Fill expands from the right side. Reading-direction aware. Feels like the next step is just past the button.

Shadow lift

The button rises off the page via translateY + shadow deepening. Creates a tactile, pressable feel without 3D borders.

Glow ring

A gold outline ring appears at a distance from the button. Creates an aura effect. Use for moments that need brand warmth.

Scale grow

Subtle scale increase (1.05x) with shadow deepening. Feels responsive and springy. Common in more playful product interfaces.

Border to gold

Faint border animates to accent color. Text shifts to match. Minimal, elegant. Works for secondary actions that shouldn't shout.

Underline reveal

A gold underline expands outward from center. For text-only actions that want just enough hover feedback to feel intentional.

Icon Placement

Icon left

Icon precedes the label. Reinforces meaning before the word. Good for directional or action-type verbs.

Icon right

Icon follows the label. The word lands first, then the icon adds meaning. Use for result-type actions like download or export.

Icon only — square

No label. Requires aria-label. Square shape fits toolbar grids. Hover state must be unambiguous.

Icon only — circle

Circular shape draws more attention than square. FAB (floating action button) patterns use this to anchor primary creation actions.

Icon + outlined

Icon reinforces an outlined secondary button. The lighter fill lets the icon carry proportionally more visual weight.

States

Default

The resting state. Full opacity, normal shadow. This is the baseline every other state is evaluated against.

Loading

Spinner replaces or precedes label. Reduced opacity and pointer-events: none prevent double-submission. The spinner is the feedback.

Success

Green background with checkmark confirms completion. Transient state — revert to default after 2 seconds or on next interaction.

Disabled

30% opacity, no pointer events. The button shape remains in the layout — don't hide it. Ideally explain why it's disabled near it.

Destructive

Red signals danger. Never use red for anything other than irreversible or high-risk actions or users will ignore the signal entirely.

  • Section Heading

    Major section breaks within a page. Establishes the second tier of hierarchy used for feature groups, content modules, and topic clusters. Visually subordinate to H1 but still commands attention.

  • Subsection Title

    Scale

    Third-level grouping for card titles, sidebar heads, and nested content. Often in medium-weight sans-serif to visually differentiate from the serif H1/H2 above it.

  • The best typography is invisible it serves the content without drawing attention to itself. Comfortable line-height and generous measure make long passages easy to read without fatigue.

    Scale

    Body Text

    The workhorse of UI text (15–18px). Optimized for sustained reading with generous line-height (1.5–1.7) and measure of 60–75 characters per line. Never go below 14px for paragraphs.

    Scale

    Caption / Label

    Smallest intentional type size (10–12px). Uppercase with wide tracking (0.2–0.3em) for overlines and UI labels; sentence case for image captions and form hints. Never use for primary information.

    Weight

    Font Weight (400 / 500 / 700)

    Weight creates emphasis within the same typeface. Use a maximum of three weights per design. Reserve bold exclusively for the most critical words overuse flattens the hierarchy and loses impact.

  • Playfair Display Serif Inter Sans-serif JetBrains Mono Monospace

    Family

    Font Family (Serif / Sans / Mono)

    Pairing a high-contrast serif (editorial, expressive) with a humanist sans (legible, neutral) creates visual depth. Monospace is reserved for code, data, and technical strings where character distinction matters.

  • Tight 1.2 This text has cramped vertical space. Reading becomes tiring over longer passages.
    Relaxed 1.7 This text breathes. The eye moves naturally from line to line without effort.

    Rhythm

    Line Height

    Vertical space between baselines. Body text needs 1.5–1.7 for readability; headlines use 0.9–1.1 for drama. Unitless values (e.g. 1.625) scale automatically with font size always prefer over fixed px values.

  • −0.02em Tight Display headlines only 0em Default Body and UI text 0.1em Wide Subheadings 0.28em Ultra Labels & overlines

    Rhythm

    Letter Spacing / Tracking

    Tight tracking on large headlines adds compression and drama. Wide tracking on small uppercase labels increases legibility at small sizes and conveys refinement. Never adjust tracking on lowercase body text.

    A sequential set of tints through shades (50–900) for a single hue. Provides consistent palette options for backgrounds, borders, text, hover, and active states eliminating ad-hoc color decisions.

    Theory

    Hue / Tint / Shade

    Hue is the base color at full saturation. Adding white creates a tint (lighter). Adding black creates a shade (darker). Adding grey creates a tone (desaturated). Building this way ensures harmonious, predictable relationships.

    Contrast Ratio

    Theory

    Color Temperature

    Warm hues (red, orange, yellow) advance visually and feel energetic, inviting, and urgent. Cool hues (blue, green, violet) recede and feel calm, trustworthy, and professional. Neutral greens and earth tones sit between and work universally. Brand personality often starts here.

    Theory

    Saturation

    Saturation is the intensity of a hue. Highly saturated colors feel energetic, playful, and consumer-facing. Desaturated (muted) colors read as sophisticated, premium, and restrained. Most luxury brands run deliberately low saturation palettes — brightness and purity feel cheap at high price points.

    Strategy

    The 60-30-10 Rule

    A palette proportion rule: 60% dominant (usually neutral — background, base surfaces), 30% secondary (supporting tone — headers, cards, UI structure), 10% accent (the brand's signature color — buttons, highlights, interactive states). Breaking this ratio is how you signal something is wrong or urgent.

    Strategy

    Industry Color Conventions

    Certain hues carry strong sector associations built up over decades. Blue signals trust (finance, tech, healthcare). Green signals nature and growth. Black signals luxury and precision. These aren't rules to follow blindly — breaking convention can differentiate a brand — but they're invisible expectations your audience arrives with.

  • BrandAccentNeutralBase

    Strategy

    Building a Balanced Palette

    A complete UI palette needs four layers: a brand color (primary actions, key UI moments), an accent (hover states, highlights — often a complementary hue), a neutral scale (backgrounds, surfaces, borders, secondary text), and functional colors (success, warning, danger). The brand color is the only one that should feel distinctive. Everything else should feel invisible.

    System

    The Spacing Scale

    A consistent set of space values multiples of 4 or 8 from which all padding, margins, and gaps are drawn. Eliminates arbitrary magic numbers and creates a visual rhythm the eye can follow across the entire interface.

  • Padding inside
    Gap between

    Model

    Padding vs Margin / Gap

    Padding is internal space between a container's edge and its children. Gap (CSS gap property) is external space between siblings in flex or grid. Modern gap is preferred over margin hacks: it doesn't collapse and works bidirectionally.

  • First paragraph with proper line-height establishing the vertical cadence of the block.

    Second paragraph the gap between them equals exactly one line-height unit, maintaining the beat.

    Third paragraph. The eye anticipates consistent spacing and reads faster when that expectation is met.

    Rhythm

    Vertical Rhythm

    A system where all vertical spacing line-height, paragraph gaps, section breaks derives from a single baseline unit. Creates a reading cadence the eye can lock into, making even dense layouts feel calm and organized.

  • Control

    Text Input

    The foundational form control for single-line text. Always pair with a visible label above the field placeholder text disappears when the user types. The focus state must be visually distinct from the default state.

  • Control

    Textarea

    Multi-line text entry for open-ended input. Set a sensible initial height (rows attribute) and allow vertical resize. Fix the width to prevent layout-breaking resizes from users dragging the handle.

  • Control

    Select / Dropdown

    Native select for 5–15 mutually exclusive options. Accessible by default a significant advantage over custom replacements. For searchable or fully styled dropdowns, build a custom combobox with proper ARIA roles.

  • Control

    Radio Buttons

    Mutually exclusive selection from a visible set (2–7 options). All choices are visible simultaneously better than a select for comparison. Group with fieldset/legend. Use when users need to see all options before deciding.

    Input States

    Every input must communicate four states clearly: default (resting), focus (active, distinct border), error (invalid with an explanatory message, not just color), and disabled (unavailable, muted). Color alone is never sufficient always add a secondary signal.

Visual Reference

Input Style Variants

Underline only

Bottom border only. Minimal, editorial. Merges with the page — the label and text carry all the weight. Best on light, clean backgrounds.

Outlined (box)

Full border on all sides. The field is clearly a container. Works on any background color and is the most universally recognized input pattern.

Filled (tinted bg)

Tinted background with only a bottom border. No border box. Common in Material Design. High affordance — the fill clearly signals an interactive zone even before focus.

Affixes — Icon & Button Attachments

Leading icon

Icon precedes the field. Reinforces the input's purpose before the user reads the placeholder. Search, location, and email icons are most common.

Trailing icon

Icon follows the field. Used for action indicators (clear X, toggle visibility, copy to clipboard). Should be actionable, not purely decorative.

Trailing button

A button fused to the right edge of the field. Use for search submit, URL copy, coupon apply, or subscribe. The single-field form pattern.

Floating Label

Resting state

Label sits inside the field as a placeholder. On focus or when filled, it animates up and shrinks to a small label above. Click the field to see it.

Filled / active state

Label floats above the field, styled as a small uppercase label in accent color. The field and its label now occupy the same vertical space.

Size Variants

Compact — 13px

Small type, minimal padding. Dense forms, inline editable tables, and settings panels.

Standard — 15px

The default. Works on every device and use case. The safe choice for any form that isn't trying to be compact or bold.

Large — 18px

Hero sign-up fields, waitlist inputs, and prominent search bars. The large size signals that this field is the primary action on the page.

Helper Text & Character Count

Minimum 8 characters. Include a number and a symbol.

Helper text below

Guidance appears below the field before and during input. Reduces errors proactively. Never replace this with placeholder text.

Inline error message

Error message replaces or appears below helper text. Must describe what's wrong and how to fix it. Color alone is not enough.

0 / 80

Character counter

Live count of characters entered against a max. Appears when the limit is meaningful to the user. Turns red near the limit.

Grouped Inputs

Horizontal group

Two fields side by side with a shared baseline. Groups related inputs that belong together conceptually: first + last name, city + zip, day + month + year.

Vertical stack

Stacked fields with consistent gap. The default form layout. Each field gets full width and equal visual treatment. Clear reading order from top to bottom.
